## Overview

| Store       | Purpose                                                                     | Provider (prod)                     |
| ----------- | --------------------------------------------------------------------------- | ----------------------------------- |
| CockroachDB | Persistent time-series storage for metrics, alerts, and windowed aggregates | CockroachDB Serverless (Basic plan) |
| Redis       | Short-lived cache for real-time dashboard reads                             | Redis Cloud (30 MB free tier)       |

Connection is configured via environment variables. See `.env.example` for the full list. The `DATABASE_URL` variable is consumed directly by dbmate for migrations; all application code derives its DSN from `POSTGRES_*` variables via `src/config.py`.

## CockroachDB — Table Schemas

> **Note:** CockroachDB requires a primary key on every table. DDL statements run outside transactions (`transaction:false` in dbmate) because CockroachDB does not support DDL inside explicit transactions.

---

### orderbook_metrics

Stores every processed order book metric emitted by the Flink `orderbook_metrics` job. This is the highest-volume table.

```sql
CREATE TABLE orderbook_metrics (
  time                TIMESTAMPTZ   NOT NULL,
  symbol              STRING        NOT NULL,
  mid_price           FLOAT8        NOT NULL,
  best_bid            FLOAT8        NOT NULL,
  best_ask            FLOAT8        NOT NULL,
  imbalance_ratio     FLOAT8,                 -- (bid_vol - ask_vol) / total_vol, range [-1, 1]
  weighted_imbalance  FLOAT8,                 -- distance-weighted imbalance, range [-1, 1]
  bid_volume          FLOAT8,
  ask_volume          FLOAT8,
  total_volume        FLOAT8,
  spread_bps          FLOAT8,                 -- (best_ask - best_bid) / mid_price * 10000
  spread_abs          FLOAT8,                 -- best_ask - best_bid
  vtob_ratio          FLOAT8,                 -- (top_bid_vol + top_ask_vol) / total_vol
  best_bid_volume     FLOAT8,
  best_ask_volume     FLOAT8,
  imbalance_velocity  FLOAT8,                 -- rate of change in imbalance_ratio (nullable)
  depth_levels        INT8,                   -- number of price levels used in calculation
  update_id           INT8,                   -- Binance exchange update ID

  CONSTRAINT pk_metrics PRIMARY KEY (symbol, time DESC)
);
```

**Conflicts:** On duplicate `(symbol, time)`, all metric columns are overwritten with `EXCLUDED` values.

**TTL:** 7 days (`time + INTERVAL '7 days'`), job runs `@hourly`.

**Data source:** Written by `DatabaseConsumer` (`src/consumers/db_consumer.py`) consuming from the `orderbook.metrics` Redpanda topic. Metrics are calculated by the Flink `orderbook_metrics.py` job from raw Binance WebSocket snapshots.

### orderbook_alerts

Stores triggered alerts. Lower volume than `orderbook_metrics` — alerts are inserted immediately (not batched) by the consumer.

```sql
CREATE TABLE orderbook_alerts (
  id               INT8      PRIMARY KEY DEFAULT unique_rowid(),
  time             TIMESTAMPTZ NOT NULL,
  symbol           STRING      NOT NULL,
  alert_type       STRING      NOT NULL,   -- EXTREME_IMBALANCE | IMBALANCE_FLIP | SPREAD_WIDENING | VELOCITY_SPIKE | UNUSUAL_VOLUME
  severity         STRING      NOT NULL,   -- LOW | MEDIUM | HIGH | CRITICAL
  message          STRING,
  metric_value     FLOAT8,                 -- the value that triggered the alert
  threshold_value  FLOAT8,                 -- the threshold that was crossed
  side             STRING,                 -- BID | ASK (nullable)
  mid_price        FLOAT8,
  imbalance_ratio  FLOAT8
);
```

**No TTL** — alerts are retained indefinitely (no storage pressure given low volume).

**Data source:** Written by `DatabaseConsumer` consuming from the `orderbook.alerts` Redpanda topic. Alerts are generated by the Flink `orderbook_alerts.py` job, which includes rate-limiting via `AlertRateLimiter` (60s cooldown per `symbol + alert_type`).

### orderbook_metrics_windowed

Stores window aggregate results produced by the Flink `orderbook_windows.py` job.

```sql
CREATE TABLE orderbook_metrics_windowed (
  time                     TIMESTAMPTZ  NOT NULL,
  symbol                   STRING       NOT NULL,
  window_type              STRING       NOT NULL,   -- '1m_tumbling' | '5m_sliding'
  window_start             TIMESTAMPTZ  NOT NULL,
  window_end               TIMESTAMPTZ  NOT NULL,
  window_duration_seconds  INT8         NOT NULL,   -- 60 or 300
  avg_mid_price            FLOAT8,
  avg_imbalance            FLOAT8,
  min_imbalance            FLOAT8,
  max_imbalance            FLOAT8,
  avg_spread_bps           FLOAT8,
  min_spread_bps           FLOAT8,
  max_spread_bps           FLOAT8,
  avg_bid_volume           FLOAT8,
  avg_ask_volume           FLOAT8,
  avg_total_volume         FLOAT8,
  total_bid_volume         FLOAT8,
  total_ask_volume         FLOAT8,
  total_volume             FLOAT8,
  sample_count             INT8         NOT NULL,   -- number of metrics ticks in the window
  window_velocity          FLOAT8,                  -- reserved, currently NULL

  CONSTRAINT pk_windowed PRIMARY KEY (time, symbol, window_type)
);
```

**Conflicts:** On duplicate `(time, symbol, window_type)`, the row is silently skipped (`DO NOTHING`).

**TTL:** 30 days (`time + INTERVAL '30 days'`), job runs `@hourly`.

**Window types:**

- `1m_tumbling` — non-overlapping 1-minute tumbling windows (`TumblingProcessingTimeWindows`)
- `5m_sliding` — 5-minute windows sliding every 60 seconds (`SlidingEventTimeWindows`)

**Data source:** Written by `DatabaseConsumer` (windowed consumer group `db-writer-windowed`) consuming from the `orderbook.metrics.windowed` Redpanda topic. Produced by the Flink `orderbook_windows.py` job.

### Views

#### latest_windowed_metrics

Returns the single most-recent window record per `(symbol, window_type)` pair. Used by the dashboard to avoid `DISTINCT ON` (not supported in CockroachDB).

```sql
CREATE VIEW latest_windowed_metrics AS
SELECT
  symbol,
  window_type,
  time,
  window_start,
  window_end,
  EXTRACT(EPOCH FROM (window_end - window_start))::INT AS window_duration_seconds,
  avg_mid_price,
  avg_imbalance,
  avg_spread_bps,
  avg_total_volume,
  sample_count
FROM orderbook_metrics_windowed w
WHERE time = (
  SELECT MAX(time)
  FROM orderbook_metrics_windowed
  WHERE symbol = w.symbol AND window_type = w.window_type
);
```

> **CockroachDB note:** `DISTINCT ON` is not supported. This correlated subquery pattern is the approved workaround. See commented-out alternative in `20260301000100_baseline_schema.sql`.

### TTL Policies

CockroachDB row-level TTL is managed via `ALTER TABLE ... SET (ttl_expiration_expression, ttl_job_cron)`. This is set in migration `20260301000200_retention_ttl.sql`.

| Table                        | Retention  | Expression                  |
| ---------------------------- | ---------- | --------------------------- |
| `orderbook_metrics`          | 7 days     | `time + INTERVAL '7 days'`  |
| `orderbook_metrics_windowed` | 30 days    | `time + INTERVAL '30 days'` |
| `orderbook_alerts`           | indefinite | —                           |

To remove TTL (rollback):

```sql
ALTER TABLE orderbook_metrics RESET (ttl_expiration_expression, ttl_job_cron);
ALTER TABLE orderbook_metrics_windowed RESET (ttl_expiration_expression, ttl_job_cron);
```

## Redis — Key Structure

Redis is used as a short-lived read cache for the Streamlit dashboard. All keys are written by `RedisConsumer` (`src/consumers/redis_consumer.py`) consuming from Redpanda. The client is defined in `src/common/redis_client.py`.

All values are JSON-serialized strings. Key naming convention is `orderbook:{scope}:{identifier}`.

---

### Key Reference

| Key Pattern                                                     | Type       | TTL                                | Written by               | Description                                                                                                                                                                                 |
| --------------------------------------------------------------- | ---------- | ---------------------------------- | ------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `orderbook:{SYMBOL}:latest`                                     | String     | 60s                                | `_handle_cache_metrics`  | Most recent processed metrics for a symbol. Payload mirrors `orderbook_metrics` columns plus `timestamp`.                                                                                   |
| `orderbook:{SYMBOL}:snapshot`                                   | String     | `max(30s, MIN_WRITE_INTERVAL * 2)` | `_handle_cache_snapshot` | Latest raw order book snapshot (bids/asks arrays + timestamp). Write rate is throttled by `REDIS_SNAPSHOT_MIN_WRITE_INTERVAL_SECONDS` (default 60s) to stay within free-tier write budgets. |
| `orderbook:alerts:{SYMBOL}`                                     | Sorted Set | 3600s                              | `_handle_add_alert`      | Per-symbol alert history. Score = alert Unix timestamp. Capped at 100 entries.                                                                                                              |
| `orderbook:alerts:ALL`                                          | Sorted Set | 3600s                              | `_handle_add_alert`      | Cross-symbol alert feed. Same structure as per-symbol set.                                                                                                                                  |
| `orderbook:stats:{SYMBOL}`                                      | String     | 300s                               | `cache_statistics`       | Windowed stats (default/legacy key). Payload includes `window_type`, `avg_imbalance`, `avg_spread_bps`, etc.                                                                                |
| `orderbook:stats:1m:{SYMBOL}`                                   | String     | 300s                               | `cache_statistics`       | Stats keyed to `1m_tumbling` window type.                                                                                                                                                   |
| `orderbook:stats:5m:{SYMBOL}`                                   | String     | 300s                               | `cache_statistics`       | Stats keyed to `5m_sliding` window type.                                                                                                                                                    |
| `orderbook:windowed:{WINDOW}:{SYMBOL}`                          | Sorted Set | 3600s                              | `add_windowed`           | History of windowed aggregate payloads. Score = `window_end` Unix timestamp. Trimmed to last 100 windows.                                                                                   |
| `orderbook:windowed:5m_sliding:{SYMBOL}:volatility:{TZ}:{DAYS}` | String     | 60s                                | `insert_volatility_data` | Pre-computed volatility series for dashboard charts. Keyed by timezone and day range.                                                                                                       |

**Symbol format:** Always uppercase (e.g., `BTCUSDT`, `ETHUSDT`, `SOLUSDT`, `BNBUSDT`).

**Write budget note:** The `REDIS_SNAPSHOT_MIN_WRITE_INTERVAL_SECONDS=60` default limits snapshot writes to ~43,200 per symbol per month. Combined with 60s dashboard read intervals this keeps total operations well under the Redis Cloud free tier limit of 100k/month.

## Data Sources & Flow

```
Binance WebSocket (wss://stream.binance.com:9443/ws)
    │  Partial depth stream: {symbol}@depth20@100ms
    ▼
IngestionService (src/ingestion/main.py)
    │  Parses → OrderBookSnapshot → publishes
    ▼
Redpanda: orderbook.raw
    │  3 partitions, 1h retention
    ▼
Flink: orderbook_metrics.py          Flink: orderbook_alerts.py       Flink: orderbook_windows.py
    │  calculate_metrics()               │  ImbalanceFlipDetector           │  WindowAggFunction
    │  → orderbook.metrics               │  AlertRateLimiter (60s)           │  1m tumbling + 5m sliding
    ▼                                    │  → orderbook.alerts               ▼
Redpanda: orderbook.metrics          ▼                              Redpanda: orderbook.metrics.windowed
    │  3 partitions, 1h retention    Redpanda: orderbook.alerts          │  3 partitions, 1h retention
    │                                    │  1 partition, 24h retention       │
    ▼                                    ▼                                   ▼
DatabaseConsumer (group: db-writer-metrics)              DatabaseConsumer (group: db-writer-windowed)
RedisConsumer    (group: redis-writer)
    │                                                         │
    ▼                                                         ▼
CockroachDB: orderbook_metrics                        CockroachDB: orderbook_metrics_windowed
CockroachDB: orderbook_alerts                         Redis: orderbook:windowed:*
Redis: orderbook:{symbol}:latest
Redis: orderbook:{symbol}:snapshot
Redis: orderbook:alerts:*
Redis: orderbook:stats:*
    │
    ▼
Streamlit Dashboard (reads Redis first, falls back to CockroachDB)
```

**Configured symbols** (via `SYMBOLS` env var): `BTCUSDT`, `ETHUSDT`, `SOLUSDT`, `BNBUSDT`

**Depth levels used in metric calculation**: controlled by `CALCULATE_DEPTH` (default `10`), independent of `DEPTH_LEVELS` (default `20`) which controls how many levels are fetched from Binance.

## Managing Migrations (dbmate)

Schema migrations live in `db/migrations/`. The canonical schema snapshot is `db/schema.sql`, which dbmate regenerates automatically on each `up`.

**Tool:** [dbmate](https://github.com/amacneil/dbmate) (`amacneil/dbmate:2` Docker image)

**Important:** All migrations use `-- migrate:up transaction:false` because CockroachDB requires DDL statements to run outside of explicit transactions.

### Migration Files

| File                                            | Description                                                                                                                |
| ----------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
| `20260301000100_baseline_schema.sql`            | Creates `orderbook_metrics`, `orderbook_alerts`, `orderbook_metrics_windowed`, and the `latest_windowed_metrics` view      |
| `20260301000200_retention_ttl.sql`              | Adds TTL policies — 7 days on `orderbook_metrics`, 30 days on `orderbook_metrics_windowed`                                 |
| `20260302000100_add_avg_mid_price_windowed.sql` | Adds `avg_mid_price FLOAT8` column to `orderbook_metrics_windowed`; recreates `latest_windowed_metrics` view to include it |

### Adding a New Migration

```bash
# Create a new migration file (timestamp is auto-generated)
dbmate new <migration_name>

# Example
dbmate new add_index_on_severity
```

This creates `db/migrations/{timestamp}_add_index_on_severity.sql`. Add your SQL under `-- migrate:up transaction:false` and the rollback under `-- migrate:down transaction:false`.

### Deployment

Migrations run automatically on deploy via the `db-migrate` Docker service before `ingestion` and `consumers` start:

```yaml
# compose.oci.yml
db-migrate:
  image: amacneil/dbmate:2
  command: ['--migrations-dir', '/db/migrations', 'up']
```

The `consumers` service has `depends_on: db-migrate: condition: service_completed_successfully`, so it will not start until migrations finish.

## Helpful Commands

### dbmate

```bash
# Apply all pending migrations
dbmate --migrations-dir db/migrations up

# Check migration status
dbmate --migrations-dir db/migrations status

# Roll back the last migration
dbmate --migrations-dir db/migrations down

# Create a new migration
dbmate new <name>

# Dump current schema to db/schema.sql
dbmate dump

# Run via Docker (matches prod behavior)
docker run --rm \
  -e DATABASE_URL="postgresql://orderbook_user:orderbook_pass@localhost:26258/orderbook?sslmode=disable" \
  -v $(pwd)/db/migrations:/db/migrations \
  amacneil/dbmate:2 up
```

### CockroachDB (local docker-compose)

```bash
# Open SQL shell
docker exec -it orderbook_cockroachdb \
  cockroach sql --insecure --host=localhost:26258

# Check cluster status
docker exec -it orderbook_cockroachdb \
  cockroach node status --insecure --host=localhost:26258

# Count rows per table
SELECT 'orderbook_metrics' AS tbl, COUNT(*) FROM orderbook_metrics
UNION ALL SELECT 'orderbook_alerts', COUNT(*) FROM orderbook_alerts
UNION ALL SELECT 'orderbook_metrics_windowed', COUNT(*) FROM orderbook_metrics_windowed;

# Check TTL job status
SELECT * FROM [SHOW JOBS] WHERE job_type = 'ROW LEVEL TTL';

# Check most recent metric per symbol
SELECT DISTINCT ON (symbol) symbol, time, mid_price, imbalance_ratio
FROM orderbook_metrics
ORDER BY symbol, time DESC;

# Check latest windowed metrics via view
SELECT * FROM latest_windowed_metrics;

# Check table sizes
SELECT
  table_name,
  pg_size_pretty(pg_total_relation_size(quote_ident(table_name))) AS size
FROM information_schema.tables
WHERE table_schema = 'public';

# Check active TTL settings
SELECT
  table_name,
  reloptions
FROM pg_class
JOIN pg_namespace ON pg_namespace.oid = pg_class.relnamespace
WHERE pg_namespace.nspname = 'public'
  AND reloptions IS NOT NULL;
```

### Redis

```bash
# Connect to Redis (local)
redis-cli -h localhost -p 6379

# List all orderbook keys
redis-cli --scan --pattern "orderbook:*"

# Get latest metrics for a symbol
redis-cli get "orderbook:BTCUSDT:latest"

# Get cached snapshot
redis-cli get "orderbook:BTCUSDT:snapshot"

# Get most recent alerts for a symbol (last 10)
redis-cli zrevrange "orderbook:alerts:BTCUSDT" 0 9 withscores

# Get windowed metrics (most recent 5)
redis-cli zrevrange "orderbook:windowed:5m_sliding:BTCUSDT" 0 4

# Check TTL remaining on a key
redis-cli ttl "orderbook:BTCUSDT:latest"

# Count all orderbook keys
redis-cli --scan --pattern "orderbook:*" | wc -l

# Clear all cached data for a symbol
redis-cli --scan --pattern "orderbook:*:BTCUSDT*" | xargs redis-cli del

# Flush all (dev/test only — DANGEROUS)
redis-cli flushall

# Monitor live writes
redis-cli monitor | grep orderbook
```
